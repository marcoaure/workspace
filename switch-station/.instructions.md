# Switch Station — Contexto para Claude/Copilot

## O que é este projeto

Ferramenta para trocar estação de trabalho dual-monitor entre Windows PC e MacBook via DDC/CI. Um comando troca os 2 monitores de input. Periféricos Logitech trocam via Easy Switch manual.

## Hardware atual

- **Monitor 1:** Alienware AW2725DF 27" QD-OLED — DP-1 (Windows) / DP-2 (Mac)
- **Monitor 2:** Samsung UR550 (LU28R55) 28" 4K — DP (Windows) / HDMI (Mac)
- **Teclado:** Logitech MX Keys (canal 1 = Win, canal 2 = Mac)
- **Mouse:** Logitech MX Anywhere 3S (canal 1 = Win, canal 2 = Mac)
- **GPU:** AMD Radeon RX 6900 XT
- **Mac:** Apple Silicon MacBook

## Valores DDC/CI (VCP 0x60) — CRÍTICO

### Alienware AW2725DF
- Windows (DP-1): **15** (0x0F)
- Mac (DP-2): **19** (0x13)
- Quirk: read retorna valores de 2 bytes (3855 = 0x0F0F para DP-1, 3859 = 0x0F13 para DP-2). O valor real é o low byte.

### Samsung UR550
- Windows (DP): **15** (0x0F)
- Mac (HDMI): **5** (0x05) — **NÃO É O PADRÃO MCCS!**
- Quirk: reads são inconsistentes (retorna 0, 5, 6, ou 15 aleatoriamente). Writes funcionam normalmente.
- **NUNCA usar valores padrão MCCS (17/18) para HDMI no Samsung** — eles piscam e voltam.
- Samsung usa: HDMI-1 = 5, HDMI-2 = 6, DP = 15

## Match patterns no discover

**Windows (DDC/CI via dxva2.dll):**
- Alienware aparece como `"Alienware AW2725DF(DisplayPort1)"` → match_windows: `Alienware|AW2725`
- Samsung aparece como `"Generic PnP Monitor"` → match_windows: `Generic PnP`

**Mac (m1ddc display list):**
- Alienware aparece como `"AW2725DF"` → match_mac: `AW2725`
- Samsung aparece como `"LU28R55"` → match_mac: `LU28R55`
- Display numbers: Alienware = 1, Samsung = 2

## Tecnologia

- **Windows:** PowerShell + P/Invoke para `dxva2.dll` (API nativa, zero dependências externas)
- **Mac:** `m1ddc` para monitores via DisplayPort + `BetterDisplay CLI` para monitores via HDMI (m1ddc não suporta DDC/CI via HDMI no Apple Silicon)
- **Hotkeys Mac:** Hammerspoon (⌃⌥W / ⌃⌥M) — config em `~/.hammerspoon/init.lua`
- **Protocolo:** DDC/CI VCP code 0x60 (Input Source Select) enviado pelo cabo de vídeo

## Regras para modificações

1. **Ao trocar monitor ou cabo:** rodar `discover.ps1` (Win) ou `discover.sh` (Mac) para verificar novos valores. Atualizar apenas `config.json`.
2. **Se um valor de input não funcionar:** testar brute-force de 1-20 no VCP 0x60. Muitos Samsung/LG não seguem o padrão MCCS.
3. **Nunca usar ControlMyMonitor** — trava/timeout. Usar sempre a DDC.psm1 via dxva2.dll.
4. **config.json é a fonte de verdade única** para ambos Windows e Mac. Mac lê config.json via jq.
5. **Dependências Mac:** m1ddc (DP) + BetterDisplay CLI (HDMI) + jq (todos instalados automaticamente pelo setup.sh).
6. **mac_tool no config.json:** define qual ferramenta usar por monitor — `m1ddc` para DP, `betterdisplay` para HDMI. BetterDisplay precisa estar rodando.

## Arquivos-chave

| Arquivo | Propósito |
|---|---|
| `config.json` | Mapeamento de inputs (fonte de verdade única — Win + Mac) |
| `lib/DDC.psm1` | Módulo DDC/CI core (dxva2.dll P/Invoke) |
| `switch-to-mac.ps1` | Script de troca principal Windows (suporta -Reverse) |
| `mac/switch-to-windows.sh` | Script de troca no Mac (lê config.json via jq) |
| `mac/setup.sh` | Setup completo Mac (instala m1ddc, jq, BetterDisplay, Hammerspoon, aliases, hotkeys) |
| `discover.ps1` / `mac/discover.sh` | Descoberta de monitores e valores |
